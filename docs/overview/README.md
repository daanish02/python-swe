# Overview

## About This Section

This section provides the foundational context for learning Python as a software engineering tool. It covers what Python is, why it matters, how it thinks and executes, and the mental models that will guide your understanding throughout your journey. These concepts form the philosophical and conceptual foundation upon which all practical skills are built.

Understanding the "why" and "how Python thinks" before diving into syntax helps you write code that aligns with Python's design philosophy rather than fighting against it.

## Contents

### [Introduction to Python](introduction.md)

What Python is, its history and evolution, why it's worth learning, and its role in modern software engineering. Covers Python's strengths, use cases, and what makes it different from other programming languages.

### [Design Philosophy](design-philosophy.md)

The Zen of Python and core design principles that guide the language. Understanding these principles helps you write idiomatic, Pythonic code and make design decisions that align with the language's values.

### [Execution Model](execution-model.md)

How Python code actually runs - from source code to execution. Covers the compilation to bytecode, the role of the Python Virtual Machine (PVM), interpretation, and the basics of Python's runtime behavior.

### [Mental Models](mental-models.md)

Frameworks for thinking about Python programs. Covers how to think about Python's object model, namespaces, scopes, mutability, and other conceptual models that help you reason about code behavior and make better design decisions.
