# Tools and Ecosystem

## About This Section

This section covers the tools, utilities, and ecosystem knowledge needed for professional Python development. From installation and environment management to development tools and workflows, these are the practical skills for productive Python work.

Understanding the Python ecosystem helps you set up efficient development environments, manage dependencies, and leverage the right tools for different tasks.

## Contents

### [Python Setup and Installation](python-setup.md)

Installing Python, managing Python versions, system Python vs user Python, Python on different operating systems, and choosing the right Python version for your needs.

### [Development Environments](development-environments.md)

IDEs and editors (VS Code, PyCharm, Vim/Neovim), Jupyter notebooks for exploration, REPL and interactive Python, configuring your editor, and productivity tips.

### [Running Python Code](running-python.md)

Python scripts (.py files), interactive mode, REPL usage, notebooks, modules vs scripts, **name** == "**main**", and different ways to execute Python code.

Using pip, poetry, uv, requirements.txt and requirements-dev.txt, pyproject.toml and uv.lock, understanding PyPI, pip install variations, managing dependencies, pip freeze, and dependency version pinning.

### [Virtual Environments](virtual-environments.md)

Why virtual environments matter, creating and activating venv, virtualenv vs venv, conda environments, environment isolation, and per-project dependencies, uv-based virtual environments.

### [Version Control with Git](version-control.md)

Git fundamentals, commits, branches, staging and committing, viewing history, undoing changes, the three trees (working directory, staging area, repository), `.gitignore`, Git configuration, and best practices. Essential commands for tracking changes and collaborating on code.

Branching strategies and collaboration patterns - Feature Branch Workflow, GitHub Flow, Git Flow, Trunk-Based Development, and Forking Workflow. Pull request best practices, merge strategies (merge, fast-forward, squash, rebase), and choosing the right workflow for your team.

Working with GitHub, GitLab, and remote repositories. Understanding remotes, pushing and pulling, authentication (HTTPS, SSH), pull requests and merge requests, issues, tags and releases, branch protection, and collaboration workflows.

### [CI/CD](ci-cd.md)
